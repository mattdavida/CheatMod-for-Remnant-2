  CHEATMOD - MOD SUMMARY
  Game: Remnant 2
================================================================================

OVERVIEW
--------
CheatMod is a UE4SS-based Lua scripting mod for Remnant 2 that grants the
player a suite of in-game cheats through hotkeys and console commands. It
operates by directly accessing and manipulating Unreal Engine player objects
at runtime using the UEHelpers library bundled with RE-UE4SS.

The mod is focused on quality-of-life cheating: keeping the player alive,
bypassing stamina limits, leveling up on demand, boosting combat feel, and
managing inventory item quantities and weapon levels — all without needing
Cheat Engine or a save editor.

God Mode is a true toggle — it hooks ModifyIncomingDamage and zeroes all
incoming damage before it is applied. The hook registers once on first use
and the toggle flag is a Lua closure shared between the command handler and
the hook callback.

Infinite Ammo registers a permanent session hook on first use. It stays
active for the rest of the session. Restart the game to disable. If the hook
drops during a loading screen transition, re-run start_infinite_ammo.

Inventory operations share a common RunFnInPlayerInv helper (in
inventory_utils.lua) that walks the player's Inventory component, keeping
per-command logic thin. A knownItemMappings table provides friendly name
aliases for common materials (e.g. "Iron" → "Material_Iron_C").

Weapon mod boosting follows a consistent per-module pattern. Each module
captures the original stat values on first activation and always multiplies
from those defaults, so pressing F4 multiple times is safe and idempotent.
Only mods currently equipped on your weapons will be present in memory —
unequipped mods are silently skipped. Some mods (Scrapshot, RottedArrow) also
zero out NumChargesConsumedOnUse so they are effectively free to cast.

CODE STRUCTURE
--------------
The mod is organized into focused modules. main.lua is purely a wiring file
(imports, keybinds, console command handlers) with no implementation logic.

  main.lua                        Wiring — imports, hotkey binds, console handlers
  player_utils.lua                Player state — stamina, cooldowns, leveling, anim speed
  inventory_utils.lua             Inventory traversal, item editing, weapon level setting
  cheat_hooks.lua                 God mode toggle + infinite ammo session hook
  WeaponMods/WeaponMods.lua       Aggregator — owns all weapon mod imports, exposes
                                  EnableAllWeaponMods() as a single entry point
  WeaponMods/mod_hotshot.lua        Weapon mod booster (Mod_Hotshot_C)
  WeaponMods/mod_sandstorm.lua      Weapon mod booster (Mod_Sandstorm_C)
  WeaponMods/mod_concussiveshot.lua Weapon mod booster (Mod_ConcussiveShot_C)
  WeaponMods/mod_helix.lua          Weapon mod booster (Mod_Helix_C)
  WeaponMods/mod_statisbeam.lua     Weapon mod booster (Mod_StasisBeam_C)
  WeaponMods/mod_voltaic_rondure.lua  Weapon mod booster (Mod_VoltaicRondure_C)
  WeaponMods/mod_scrapshot.lua      Weapon mod booster (Mod_Scrapshot_C)
  WeaponMods/mod_rottedarrow.lua    Weapon mod booster (Mod_RottedArrow_C)

Planned weapon mod additions (tracked in WeaponMods.lua):
  Mod_Skewer_C, Mod_Defrag_C, Mod_AbrasiveRounds_C, Mod_HeatWave_C, Mod_Tremor_C

Installation 
-------
Install ue4ss ( only tested on latest zdev - Make sure to extract to <path-to-your-game-files>\Remnant2\Binaries\Win64 )
add mod to Mods folder ex: <path-to-your-ue4ss-in-game>/Mods
enable in Mods/mods.txt ( add CheatMod: 1 )
============

REQUIREMENTS:
* UE4SS (Unreal Engine 4 Scripting System)
  Download: https://github.com/UE4SS-RE/RE-UE4SS/releases

INSTALLATION STEPS:
1. Install UE4SS:
   - Download latest UE4SS release
   - Extract to your game's Win64 folder:
     [Steam Path]\steamapps\common\Remnant2\Remnant2\Binaries\Win64\

2. Install CheatMod:
   - Download and extract mod files
   - Place in: ue4ss/Mods/CheatMod/

3. Configure:
   - Open ue4ss/mods.txt
   - Add line: CheatMod : 1

4. Launch game - mod will auto-initialize


FEATURES
--------

1. REPLENISH COOLDOWNS AND MOD POWER (Hotkey: F1)
   - Instantly resets all skill Cooldowns and fills Mod Power in a single
     keypress. Health, Ammo, and Dragon Hearts are not affected.
   - Recommended to bind to R1 (controller) or equivalent for quick access
     mid-combat.

2. INFINITE STAMINA (Hotkey: F2)
   - Locates the player's Stamina component and appends the FName tag
     "InfiniteStamina" to its ComponentTags array.
   - Note: May need to be re-applied after death or level transitions.

3. FAST PLAYER ACTIONS (Hotkey: F3)
   - Iterates all loaded AnimSequence objects and boosts the RateScale of
     player animations that are not standard locomotion (walk, jog, sprint,
     run, crouch, aim).
   - Evade roll is tuned to a separate value for better feel.
   - Provides a noticeably snappier combat feel without breaking traversal.

4. BOOST ALL SUPPORTED WEAPON MODS (Hotkey: F4)
   - Fires all eight weapon mod boost modules in sequence. Each module finds
     its target UE class in memory and applies stat multipliers to it.
   - Only mods currently equipped on your weapons will be present in memory;
     unequipped mods are not found and are silently skipped with no errors.
   - Each module captures baseline values on first activation — pressing F4
     again is idempotent and will not compound the multipliers.
   - Re-press after switching weapon mods to apply boosts to newly equipped ones.
   - Supported mods and their boosted stats:
       HotShot         FireDuration, FireBaseDamage, ModDuration
       Sandstorm        CycloneDuration, CycloneBaseRadius, CycloneHomingRadius,
                        CycloneDPS, CycloneDamageFrequency
       ConcussiveShot   BlastDamage, MaxRange, BaseKnockbackDistance,
                        AOERadius, MaxCharges
       Helix            MaxCharges, ImpactDamage, SideWinderDamage, SideWinderCount
       StatisBeam       Damage, Duration, RequiredHitDuration
       VoltaicRondure   PulseDelay (pinned to minimum), OrbDamage,
                        ProjectileLifetime, EffectRadius, ShockDamage, ShockDuration
       Scrapshot        MaxCharges, BlastRadius, DOTDamage, CaltropDuration,
                        BleedDamage, BleedDuration,
                        NumChargesConsumedOnUse (zeroed — free to cast)
       RottedArrow      MaxCharges, WeakSpotMod, ImpactDamage, DOTDamage,
                        CloudDuration, BlastRadius, CloudDamagePerSecond,
                        NumChargesConsumedOnUse (zeroed — free to cast)

5. LEVEL UP (Console Command: levelup [n])
   - Calls OnLevelUp() on the player's Traits object.
   - Accepts an optional numeric parameter to level up multiple times at once
     (e.g., "levelup 5" triggers 5 consecutive level-up events).
   - Defaults to 1 level-up if no parameter is provided.

6. SET INVENTORY ITEM QUANTITY (Console Command: set_inventory_item_quantity <itemName> <quantity>)
   - Iterates the player's inventory directly via the Inventory component and
     finds the first item whose blueprint name matches <itemName>.
   - Supports a knownItemMappings alias table so friendly names can be used
     (e.g. "Iron" resolves to "Material_Iron_C"); falls back to a raw name
     search if no alias is defined.
   - Sets that item's InstanceData.Quantity to <quantity> and logs both the
     old and new values.

7. SET ALL WEAPON LEVEL (Console Command: set_all_weapon_level <level>)
   - Iterates the player's inventory and upgrades every item whose blueprint
     path contains "Weapon" to the specified level.
   - Useful for quickly maxing out all carried weapons without visiting the
     upgrade bench.

8. LOG INVENTORY ITEMS (Console Command: log_inventory_items [true|false])
   - Debug/discovery command that walks the player inventory and prints item
     blueprint names and instance data to the console.
   - Without the "true" flag (default): logs only materials and consumables.
   - With "true": logs every item in the inventory regardless of type.
   - Handy for identifying the exact name to pass to set_inventory_item_quantity.

9. GOD MODE (Console Command: toggle_god_mode)
   - Toggles God Mode on/off. Hooks into ModifyIncomingDamage on first use —
     all incoming damage is zeroed before it is applied, making the player
     effectively unkillable with no visible health drop.
   - The hook registers once on first toggle and persists; subsequent calls
     toggle the damage-zeroing behavior on/off via a shared closure flag.
   - Note: if the hook drops during a loading screen transition, re-run
     toggle_god_mode twice (off then on) to re-register it.

10. INFINITE AMMO (Console Command: start_infinite_ammo)
    - Registers a permanent hook into Weapon_Gun_Base_C:OnFire. Every time
      the player fires, ammo is instantly replenished so magazines never run dry.
    - One-shot activation — active for the remainder of the session.
    - To disable: restart the game.
    - Note: the hook can occasionally drop during loading screen transitions.
      Re-run start_infinite_ammo if it stops working.


HOTKEYS REFERENCE
-----------------
  F1  - Replenish cooldowns and mod power
  F2  - Add infinite stamina tag to stamina component
  F3  - Boost non-locomotion player animation speed
  F4  - Boost all supported weapon mods (HotShot, Sandstorm, ConcussiveShot,
        Helix, StatisBeam, VoltaicRondure, Scrapshot, RottedArrow)


CONSOLE COMMANDS REFERENCE - Open console with ~ or F10
---------------------------
  levelup [n]                                        - Level up 1 or n times
  set_all_weapon_level <level>                       - Set all weapons in inventory to <level>
  set_inventory_item_quantity <itemName> <quantity>  - Set quantity of named item (alias or blueprint name)
  log_inventory_items [true|false]                   - Print inventory items to console (true = all items)
  toggle_god_mode                                    - Toggle god mode on/off (zeroes damage before it applies)
  start_infinite_ammo                                - Enable infinite ammo for this session (restart to disable)
